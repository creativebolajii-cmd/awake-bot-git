# PHASE 2 - WORDPRESS PUBLISHING MODULE

## Purpose

Phase 2 receives the approved JSON generated by Phase 1 and publishes it to the AwakeMovies WordPress website.

Phase 2 does not perform scraping, normalization, metadata enrichment, or content editing.

It only validates, uploads media, publishes content, and verifies the publication.

---

# INPUT

Receive the complete approved JSON from Phase 1.

The JSON is the single source of truth.

Do not modify any field.

Do not regenerate data.

Do not perform another scrape.

Always publish exactly what was approved by the user.

---

---

# PRE-PUBLISH VALIDATION

Before publishing, validate the incoming JSON.

Required for all content:

* post_type
* title
* synopsis
* poster_url
* trailer_url (if available)
* download_link (movie) or episode_links (series)

Movie requirements:

* release_year
* duration

Series requirements:

* season
* episode_range
* episode_list

If validation fails:

* Display the missing fields.
* Cancel publishing.
* Return the user to the editor.

---

# FEATURED IMAGE

Download the poster from the provided poster URL.

Upload it to the WordPress Media API.

Retrieve and store:

* media_id
* uploaded_image_url

If media upload fails:

* Stop immediately.
* Do not publish the post.

# SUCCESS VERIFICATION

Publishing is successful only if all of the following are returned:

* post_id
* post_url must be like this for series download-classroom-of-the-elite-season-1 instead of download-classroom-of-the-elite-s04-e01-e16-completed
* featured_image_id
* featured_image_url
* published status
* saved custom meta

If any verification fails:

Treat the publish operation as failed.

Do not report success.

# IMPLEMENTATION RULES

* Publish only approved content.
* Never publish incomplete data remember each as different logic in scraping
* Never modify the approved JSON.
* Never perform scraping during publishing.
* Never overwrite user-approved values.
* Upload the featured image before creating the post.
* Verify the published post before reporting success.
* Fail immediately if any required step cannot be completed.
* Return structured success and error responses for every publishing request.
